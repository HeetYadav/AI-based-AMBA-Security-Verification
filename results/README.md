# Detection Results

This directory stores all output artifacts produced by the machine learning detection pipeline.

## Performance Metrics (`*.json`)

Each simulation and detection pass generates two JSON files — one for Full Mode and one for Blind Mode.

File naming convention: `detection_results_{variant}_{mode}.json`

Key fields in each file:

| Field | Type | Description |
|---|---|---|
| `variant` | string | Trojan variant identifier (A, B, or C) |
| `mode` | string | Detection mode (`full` or `blind`) |
| `if_f1` | float | Isolation Forest F1 score |
| `svm_f1` | float | One-Class SVM F1 score |
| `scores` | array | Per-transaction anomaly scores |
| `n_total` | int | Total transactions in the trace |
| `n_trojan` | int | Ground-truth positive count |

## Visualization Plots (`*.png`)

PNG files generated by the pipeline for qualitative analysis:

| File | Description |
|---|---|
| `plot_transactions.png` | Time-series of bus activity across the full simulation |
| `plot_timeline_*.png` | Detection confidence score over the course of the simulation, per variant |

## Report Metrics (`results_report.txt`)

A plain-text snapshot of the output from `scripts/compute_report_values.py`. It contains the full confusion matrix breakdown, feature separation statistics, and address coverage data for all six variant/mode combinations. This file is a reference output; re-running the script will regenerate the same data from the source CSV traces.
